tzoffsettzoffset

DATEPART retorna o valor tzoffset (tz) como o número de minutos (com sinal).DATEPART returns the tzoffset (tz) value as the number of minutes (signed). Esta instrução retorna uma diferença de fuso horário de 310 minutos:This statement returns a time zone offset of 310 minutes:

SQL
SELECT DATEPART (tzoffset, '2007-05-10  00:00:01.1234567 +05:10');

DATEPART renderiza o valor de tzoffset da seguinte maneira:DATEPART renders the tzoffset value as follows:

  • Para datetimeoffset e datetime2 e, tzoffset retorna a diferença de tempo em minutos, em que a diferença para datetime2 é sempre de 0 minuto.For datetimeoffset and datetime2, tzoffset returns the time offset in minutes, where the offset for datetime2 is always 0 minutes.
  • Para tipos de dados que podem ser convertidos implicitamente em datetimeoffset ou datetime2, DATEPART retorna a diferença de tempo em minutos.For data types that can implicitly convert to datetimeoffset or datetime2, DATEPART returns the time offset in minutes. Exceção: outros tipos de dados de data/hora.Exception: other date / time data types.
  • Parâmetros de todos os outros tipos resultam em erro.Parameters of all other types result in an error.

Argumento smalldatetime de datesmalldatetime date Argument

Para um valor smalldatetime date, DATEPART retorna os segundos como 00.For a smalldatetime date value, DATEPART returns seconds as 00.

Padrão retornado para um datepart que não está em um argumento de dateDefault Returned for a datepart That Is Not in a date Argument

Se o tipo de dados do argumento date não tiver a datepart especificada, DATEPART retornará o padrão para essa datepart apenas quando um literal for especificado para date.If the date argument data type does not have the specified datepart, DATEPART will return the default for that datepart only when a literal is specified for date.

Por exemplo, o ano-mês-dia padrão para qualquer tipo de dados de date é 1900-01-01.For example, the default year-month-day for any date data type is 1900-01-01. Esta instrução tem argumentos de parte de data para datepart, um argumento de hora para date e ela retorna 1900, 1, 1, 1, 2.This statement has date part arguments for datepart, a time argument for date, and it returns 1900, 1, 1, 1, 2.

SQL
SELECT DATEPART(year, '12:10:30.123')  
    ,DATEPART(month, '12:10:30.123')  
    ,DATEPART(day, '12:10:30.123')  
    ,DATEPART(dayofyear, '12:10:30.123')  
    ,DATEPART(weekday, '12:10:30.123');

Se date é especificada como uma variável ou coluna de tabela e o tipo de dados dessa variável ou coluna não tem a datepart especificada, DATEPART retorna o erro 9810.If date is specified as a variable or table column, and the data type for that variable or column does not have the specified datepart, DATEPART will return error 9810. Neste exemplo, a variável @t tem um tipo de dados time.In this example, variable @t has a time data type. O exemplo falha porque o ano da parte de data é inválido para o tipo de dados time:The example fails because the date part year is invalid for the time data type:

SQL
DECLARE @t time = '12:10:30.123';   
SELECT DATEPART(year, @t);

Frações de segundoFractional seconds

Estas instruções mostram que DATEPART retorna frações de segundos:These statements show that DATEPART returns fractional seconds:

SQL
SELECT DATEPART(millisecond, '00:00:01.1234567'); -- Returns 123  
SELECT DATEPART(microsecond, '00:00:01.1234567'); -- Returns 123456  
SELECT DATEPART(nanosecond,  '00:00:01.1234567'); -- Returns 123456700

ComentáriosRemarks

DATEPART pode ser usado na lista de seleção e nas cláusulas WHERE, HAVING, GROUP BY e ORDER BY.DATEPART can be used in the select list, WHERE, HAVING, GROUP BY, and ORDER BY clauses.

DATEPART converte implicitamente literais de cadeia de caracteres como um tipo datetime2 em SQL Server 2019 (15.x)SQL Server 2019 (15.x).DATEPART implicitly casts string literals as a datetime2 type in SQL Server 2019 (15.x)SQL Server 2019 (15.x). Isso significa que DATENAME não oferece suporte ao formato YDM quando a data é transmitida como cadeia de caracteres.This means that DATENAME does not support the format YDM when the date is passed as a string. É necessário converter explicitamente a cadeia de caracteres em um tipo de datetime ou smalldatetime para usar o formato YDM.You must explicitly cast the string to a datetime or smalldatetime type to use the YDM format.
